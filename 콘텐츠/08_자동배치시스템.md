# 08. 자동배치 시스템 (System)

## 사용 도구
- **Rhino 8** — 3D 모델링 환경
- **Grasshopper** — 파라메트릭 정의
- **GHPython** — 배치 알고리즘 스크립트

## 목표
대지 형상에 따라 **용적률 250%**를 달성하는 최대 건폐율·최대 세대수 배치안을 자동 생성.

## 입력값
| 항목 | 설명 |
|---|---|
| 매스 이름 | 각 주동의 식별자 (예: Group301) |
| 층당 세대수 | 매스별 한 층의 세대수 |
| 대지 경계면 | 배치 가능 영역 (site 레이어, 폴리서피스) |
| 배치 기준선 | site::line — 도로/일조 방향 (평행·수직 회전 후보) |

## 매스 구성 레이어 (인동간격 자동 분류의 핵심)
| 레이어 | 색 | 내용 | 인동거리 |
|---|---|---|---|
| `plan` | 빨강 | 측벽 (창 없는 외벽) | 4m (측벽–측벽) |
| `plan > bay` | 파랑 | **채광창** (≥0.5㎡) | 0.5H |
| `plan > little` | 하늘 | **소창** (<0.5㎡, 채광창 없음) | 8m |
| `core` | 회색 | 코어(계단·EV) | — |

- 알고리즘이 footprint 각 엣지에 bay/little 선 중점을 **수직 투영**(거리 <500mm)해 면 유형을 자동 분류
- 분류된 면 유형으로 방향별 인동간격(pair_gap) 결정

## 재설계 파이프라인 (행렬 배치)
| 단계 | 처리 | 비고 |
|---|---|---|
| [1] 회전 결정 | rotation_mode 분기 (A:남향 / B:최다fit / C:packing) | step_orient |
| [2] 프로그램 산정 | 목표 동수 N · 층당세대수 nfs | step_program |
| [3] 행렬 배치 | rowplace 엔진으로 격자 배치 | step_place |
| [4] 목표 회복·bake | 점증 cap으로 FAR 250% 회복 후 bake | step_final |
| [5] corridor 시각화 | 인동 통로 시각화 | — |

- 공용 함수: `mass_layout_lib.py` (geom/io/edges/rule/rowplace/bake)
- 시나리오 토글: `SCENARIOS` dict (A/B/C)
- bake 레이어: `매스_{시나리오}_{건폐율}_{그룹}`

## 처리 흐름
```
라이노 입력 (site, plan::bay/little, 매스 그룹, 도로중심선)
   ↓
[1] 회전 → [2] N·nfs → [3] 행렬배치 → [4] FAR회복·bake → [5] 시각화
   ↓
bake 매스 → ③ 평가지표 스크립트가 읽어 A3~C2 연산
```

## 용적률 목표
| 우선순위 | 조건 |
|---|---|
| 1순위 | FAR **240~255%** (목표 250%) |
| 2순위 | 달성 가능한 최대 FAR |

## 최적화 목표 함수
동일 FAR 범위 내에서 `score = (층수↑, -건폐율)` 오름차순
→ **층수 최소화** 우선, 그 다음 **건폐율 최대화**.
