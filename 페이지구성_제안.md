## 2. 페이지별 상세 구성

### 01. 개요 (Hero)
- **타이틀**: 고정 용적률(250%)하 건폐율 변화에 따른 공동주택 배치안의 가로공간 성능 분석
- **부제(영문)**: Streetscape Performance of Apartment Layouts under Varying BCR at Fixed FAR 250%
- **한줄 컨셉**: *"방 안의 일조가 아니라, 단지 밖 가로를 걷는 보행자를 위한 배치"*
- 키비주얼: 매스 뷰 렌더(`3. 매스 분석/매스 뷰/Scene 1.png`) 또는 도시 단면
- 작성자 · 학과 · 지도교수 · 연도

### 02. 문제의식 (Problem)
- 기존 한국 아파트 단지: '방 안에서 내다보는 일조·프라이버시'만 챙김
- 결과: '단지 밖 가로를 걷는 보행자의 스케일·심리'를 망침
- 비교 이미지: 위압적인 고층 가로 vs 보행 친화 가로
- 자료원: `0. 주차별/0108/건축설계7_아파트(외부인/내부인).png`

### 03. 이론적 배경 (Theory)
- **Martin & March (1972)** *Urban Space and Structures*
  - 연면적·용적률 고정 시 → 기하학적 배치에 따라 외부공간 성격 완전히 달라짐
  - ① 저밀도 초고층 = 거대·황량한 광장형 / ② 고밀도 중저층 = 아늑한 중정형
- **아시하라 요시노부**: 앙각 27°/45°/60°, D/H 비율 1~2
- **Jan Gehl**: 25m 사회적 접촉, 5m 인지거리, Walkable City
- 다이어그램: `99. 중크/책 요약 다이어그램.3dm`, `중크_도시 다이어그램.ai`

### 04. 대상지 (Site)
- 위치: 잠실 / 대지면적 **72,695.1 m²**
- 사이트 역사 (1976~2024 변천): `0. 주차별/0319/사이트 역사.ai`, 연도별 pdf
- 현황 분석: 건물 높이/용도/연도, 대중교통·공원·학군
- 사이트 블록 분할, 유동인구·상권 분석
- 자료원: `0. 주차별/0316`, `0319`, `0323`, `0402`, `1. 사이트/`

### 05. 연구 방법 (Method)
- **3단계 흐름**
  ```
  ① 전환임계점 분석 → 건폐율 구간 산정
  ② 구간별 자동배치안 생성
  ③ 평가지표로 비교·평가
  ```
- **변인 설정 표**
  - 조작변인: 건폐율 (5/10/20/30/40/50%)
  - 통제변인: 대지·형상, 용적률 250%, 유닛 16종, 층고 3000mm
  - 종속변인: (1단계) 층수·동수·인동간격 / (2단계) 시각·가로 성능 지표
- 자료원: `4. 배치 분석/연구_전체흐름_코드매핑.md`

### 06. 유형 분류 (Typology)
- **매스 타입**: General(Plate/Tower/Mixed) × 기하형태(I·L·T·Y·X·ㅁ) × 변형 4단계
- **유닛 타입**: Open type(1~3side) · 세대조합 · 코어(Hall/계단실/혼합)
- **배치 타입**: 동간관계(Single/Parallel/Lattice/Stagger/Mix) · 가로관계 · 내부관계
- 16종 주동 유닛 카탈로그 (I2, I4, L22, V3, V4, X4, ㅁ3, ㅁ4, Y2, Y3 …)
- 자료원: `매스 유형화/매스, 유닛, 배치 종류 분류.md`, `2. 유닛/PDF/유닛 평면.pdf`

### 07. 시나리오 (Scenario)
| 시나리오 | 남향 | 인동간격 | 회전전략 | 효과 |
|---|:---:|:---:|---|---|
| **A** 남향+인동 | O | O | 남향 채광 최대 | 현황 (기준) |
| **B** 인동만 | X | O | 인동 포함 최다배치 | 동수↑, 가로 위압감↓ |
| **C** 인동무시 | X | X | packing 최다 | 저층고밀, 가로 성능 분석 |
- 인동간격 법규(건축법 시행령 §86) 표 포함

### 08. 자동배치 시스템 (System)
- 도구: Rhino 8 · Grasshopper · GHPython
- 파이프라인 5단계: 회전결정 → 프로그램산정 → 행렬배치 → FAR회복·bake → 시각화
- 인동간격 자동 분류 로직 (bay/little/plan 면 유형 → 0.5H / 8m / 4m)
- 처리 흐름 다이어그램
- 자료원: `4. 배치 분석/0. 배치안 생성/`, `매스(배치, 국내).md`

### 09. 결과 (Results) — 메인은 요약, 상세는 /results
- 건폐율별 대표 배치안 3D 뷰 (5/10/20/30/40/50%)
- 핵심 지표 변화 그래프 (건폐율 ↔ 동수·층수·앙각·SVF)
- **전환임계점 3구간**
  - 1구간(Optimal) 17.3% / 2구간(Compromise) 25.5% / 3구간(Invalid) 50.9%
- 자료원: `결과도출.md`, `결과도출_건폐율20%.md`, 시나리오 결과 md들

### 10. 평가지표 (Metrics) — 상세는 /metrics
- 3그룹: **A 물리적 형태 / B 가로변 보행자 인지 / C 단지 내부 환경**
- A3 높이 · A4 BCR · A5 동수 · A6 입면길이(frontage)
- B2 앙각 · B3 SVF(천공 가시율)
- C1 동간거리 · C2 D/H비
- 각 지표별 근거(아시하라·Gehl·조례) 카드형 표시
- 자료원: `4. 배치 분석/지표별/`

### 11. 결론 (Conclusion)
- 향(남향)·법규를 일부 양보(B·C) → 가로 위압감↓, 보행 친화 가로 완성
- 수치적 근거 요약 + 설계적 제안
- Walkable City 비전

## 4. 기술 스택 제안

| 옵션 | 구성 | 장단점 |
|---|---|---|
| **A. 정적 HTML/CSS/JS** (추천) | 순수 HTML + CSS + 약간의 JS | 가볍고 GitHub Pages 무료 배포, 유지 쉬움 |
| **B. Astro / 11ty** | 정적 사이트 생성기 + 마크다운 | 콘텐츠가 md라 관리 편함, 빌드 필요 |
| **C. React/Next** | 컴포넌트 기반 | 인터랙션 강하지만 과한 수준 |

> **제안: A 또는 B.** 콘텐츠 대부분이 이미 md/이미지라 정적 사이트가 적합.
> 스크롤 애니메이션·3D 뷰어가 필요하면 가벼운 라이브러리(GSAP, model-viewer)만 추가.

### 배포
- **GitHub Pages** (무료, 커스텀 도메인 가능) — 졸업 포트폴리오 URL로 공유 용이

### 에셋 변환 필요
- `.ai` → SVG/PNG 내보내기
- `.3dm` 렌더 → PNG (이미 `매스 뷰/Scene*.png` 존재)
- `.pdf` → 썸네일 이미지 또는 임베드 뷰어
- `.mp4`(분석현황) → 영상 임베드
