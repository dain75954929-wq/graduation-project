# 웹사이트 디자인 가이드 (Design System)

> 졸업설계: **고정 용적률(250%)하 건폐율 변화에 따른 공동주택 배치안의 가로공간 성능 분석**
> 작성일: 2026-06-07 · 이 문서는 HTML/CSS 구현의 단일 기준(Single Source of Truth)

---

## 0. 디자인 컨셉

### 키워드
**Architectural Drawing · Pedestrian Scale · Quiet Data**

건축 도면처럼 **여백이 넓고, 선이 얇고, 정보가 정연한** 화면.
화려한 장식 대신 **다이어그램·데이터·이미지 자체가 주인공**이 되게 한다.
연구의 메시지("거대한 매스가 아니라 걷는 사람의 스케일")를 화면의 **여백과 정렬**로 체화한다.

### 3대 원칙
1. **선(Line)으로 말한다** — 면 분할·구획은 굵은 색면이 아니라 1px 헤어라인으로.
2. **그리드를 드러낸다** — 건축 도면의 모눈처럼 정렬 구조가 은근히 보이게.
3. **위계는 타이포로** — 색을 거의 안 쓰는 대신 크기·굵기·자간으로 위계를 만든다.

## 1. 컬러 팔레트 — 완전 무채색 (Monochrome)

**포인트 색 없음.** 강조는 색이 아니라 **명도·면(fill)·선·타이포 굵기**로만 만든다.
→ 가장 건축적이고 엄격한 톤. 도면·먹선의 정밀함.

```css
:root {
  /* Base — 종이/잉크 (무채색, 미세하게 따뜻한 그레이) */
  --paper:      #F7F7F5;  /* 배경 (옅은 트레이싱지 화이트) */
  --paper-pure: #FFFFFF;  /* 카드·이미지 배경 */
  --ink:        #141414;  /* 본문 텍스트 (순흑에 가까운 먹) */
  --ink-70:     #474747;  /* 보조 텍스트 */
  --ink-40:     #8C8C8C;  /* 캡션·메타 */
  --line:       #D6D6D4;  /* 헤어라인·구분선 */
  --line-soft:  #EAEAE8;  /* 그리드 배경선 */

  /* Emphasis — 색 대신 '먹 면/먹 선'으로 강조 */
  --fill-ink:   #141414;  /* 강조 블록 배경(글자는 paper색 반전) */
  --fill-soft:  #ECECEA;  /* 강조 행·배지 옅은 배경 */
  --rule-bold:  #141414;  /* 굵은 강조선 2px */

  /* 데이터 시각화 — 건폐율 단계 순차 그레이스케일 */
  --seq-1: #1F1F1F;  /* 저건폐율(초고층) — 어둡고 무거움 */
  --seq-2: #555555;
  --seq-3: #888888;
  --seq-4: #B5B5B5;
  --seq-5: #DDDDDD;  /* 고건폐율(저층고밀) — 밝고 가벼움 */
}
```

> **무채색 강조 규칙 (색이 없으니 더 엄격하게)**
> - 위계는 오직 **크기·굵기·여백·라인**으로. 색으로 도망치지 않는다.
> - "강조"가 필요하면: ① 먹 면 반전(`--fill-ink`에 흰 글자) ② 2px 굵은 라인 ③ 굵은 웨이트(700).
> - 강조 요소는 한 화면에 **1개**만. (먹 블록이 여러 개면 도면이 시끄러워짐)
> - 건폐율 5→50% 그래프: `--seq-*` 어두움→밝음으로 "무거운 초고층 → 가벼운 저층" 직관 전달.

### 다크 변형 (시그니처 대비)
도시 단면·매스 렌더·Hero 섹션은 **먹 배경(`--ink`) + 종이색 텍스트**로 반전.
무채색이라 흑↔백 반전만으로도 강한 리듬이 생김 — 이것이 포인트색을 대신하는 주 무기.

## 2. 타이포그래피

### 폰트
```css
--font-kr: "Pretendard", -apple-system, sans-serif;     /* 국문 본문·제목 */
--font-en: "Inter", "Pretendard", sans-serif;            /* 영문·숫자·라벨 */
--font-mono: "JetBrains Mono", "D2Coding", monospace;    /* 수치·코드·도면 라벨 */
```
- 제목·본문: **Pretendard** (한글 가독성·중립성)
- 영문 부제·섹션번호·지표명: **Inter** (기하학적, 도면 라벨 느낌)
- BCR/FAR/앙각 등 **수치는 mono**로 → 표·데이터의 도면적 정밀함 강조

### 스케일 (Type Scale, 1.250 Major Third 기반)
| 토큰 | 크기 | 용도 |
|---|---|---|
| `--fs-display` | clamp(2.5rem, 6vw, 5rem) | Hero 타이틀 |
| `--fs-h1` | clamp(2rem, 4vw, 3rem) | 섹션 대제목 |
| `--fs-h2` | 1.75rem | 소제목 |
| `--fs-h3` | 1.25rem | 카드 제목 |
| `--fs-body` | 1.0625rem (17px) | 본문 |
| `--fs-small` | 0.875rem | 캡션·표 |
| `--fs-label` | 0.75rem | 섹션번호·메타 (대문자·자간 넓게) |

### 규칙
- 본문 `line-height: 1.7`, 제목 `1.2`
- 섹션 번호/라벨: `letter-spacing: 0.15em; text-transform: uppercase;` (예: `03b — APARTMENT ANALYSIS`)
- 한글 제목은 자간 `-0.02em`로 살짝 조여 단정하게
- 긴 본문 측정폭(measure): **최대 68ch**

## 3. 레이아웃 & 그리드

### 전역 그리드
- **12컬럼**, 최대 콘텐츠 폭 `--maxw: 1280px`, 좌우 거터 `--gutter: clamp(20px, 5vw, 80px)`
- 베이스 단위 **8px** (모든 간격은 8의 배수)

```css
--space-xs: 8px;  --space-sm: 16px; --space-md: 32px;
--space-lg: 64px; --space-xl: 120px; --space-2xl: 200px;
```

### 섹션 리듬
- 섹션 상하 패딩: `--space-xl`(데스크탑) / `--space-lg`(모바일)
- 섹션 사이는 **풀블리드 헤어라인**(`--line`)으로 구분 — 도면 경계처럼

### 모눈 배경 (시그니처 요소)
은은한 그리드 패턴을 배경에 깔아 "도면 위" 느낌.
```css
background-image:
  linear-gradient(var(--line-soft) 1px, transparent 1px),
  linear-gradient(90deg, var(--line-soft) 1px, transparent 1px);
background-size: 40px 40px;
/* opacity 낮게, Hero·구분 영역에만 부분 적용 */
```

### 레이아웃 패턴 (섹션 유형)
1. **풀폭 텍스트** — 도입/문제의식 (좌측 정렬, measure 제한)
2. **2단 분할** — 좌 텍스트 / 우 다이어그램 (이론·방법)
3. **갤러리 그리드** — 21단지 카드, 건폐율 배치안 (3~4열)
4. **풀블리드 이미지** — 매스 렌더·도시 단면 (가로 100vw)
5. **데이터 테이블** — 결과·지표 (mono 수치)

## 4. 컴포넌트

### 4.1 내비게이션
- 상단 고정(sticky), 높이 64px, 배경 `--paper` + 하단 헤어라인
- 좌: 프로젝트 약칭 / 우: 섹션 앵커 링크(번호 라벨식 `01 02 03…`)
- 스크롤 시 현재 섹션 라벨 하이라이트(먹색 2px 언더라인)
- 모바일: 햄버거 → 풀스크린 오버레이(모노톤)

### 4.2 섹션 헤더
```
03b ─────────────────────────  ← 번호 라벨(mono/label) + 헤어라인
아파트 사례 분석                  ← H1 (국문)
Apartment Streetscape Analysis  ← 영문 부제 (ink-40)
```

### 4.3 Hero
- 풀뷰포트(min-height: 100svh), 모눈 배경 + 키비주얼(렌더) 한 컷
- 타이틀(display) + 한 줄 컨셉 + 핵심 숫자 배지(250% / 5–50% / 16 / 10 / 21)
- 하단 스크롤 인디케이터(얇은 세로선 + 점)

### 4.4 카드 (21단지 / 사례)
- 배경 `--paper-pure`, 테두리 1px `--line`, radius 2px(거의 각짐)
- 상단 이미지(4:3) + 하단: 지역 라벨(label) / 단지명(h3) / 분류 태그
- hover: 테두리 `--ink`(1px→먹색 진하게) + 살짝 상승(translateY -4px), 그림자 최소

### 4.5 데이터 테이블
- 헤더: `--ink` 텍스트 + 하단 2px 라인, 본문 행 사이 1px `--line`
- 수치 컬럼 우정렬 + mono 폰트
- 강조 행(임계점·대표값)은 `--fill-soft` 배경 + 좌측 먹색 2px 바
- zebra 줄무늬 대신 **여백과 라인**으로 구분 (도면식)

### 4.6 배지 / 태그
- 배지(숫자): 테두리 1px, mono, 둥근 사각
- 시나리오 태그 A/B/C (무채색, 양보 정도를 면처리로): A=먹면 채움(fill-ink) / B=먹선 외곽선만(outline) / C=점선 외곽선(dashed)

### 4.7 다이어그램 캡션
- 이미지 하단 좌측, `--fs-small`, `--ink-40`, 앞에 `Fig.` 라벨(mono)

## 5. 모션 / 인터랙션

### 원칙
**조용하게.** 과한 패럴랙스·바운스 금지. 등장은 페이드+살짝 상승만.

```css
--ease: cubic-bezier(0.22, 1, 0.36, 1);  /* 부드러운 감속 */
--dur: 600ms;
```
- 스크롤 진입 시: `opacity 0→1`, `translateY 24px→0`, stagger 80ms
- 헤어라인/언더라인: 좌→우로 그려지듯(`scaleX`) 등장 → 도면 작도 느낌
- 숫자 배지: 카운트업(250%, 21단지 등)
- 호버 트랜지션 `200ms`
- `prefers-reduced-motion` 존중 → 모션 제거, 즉시 표시

### 시그니처 인터랙션 (욕심내면, 1개만)
- **건폐율 슬라이더**: 5→50% 드래그 시 배치안 이미지/동수·층수·앙각 수치가 실시간 전환.
  연구의 핵심(건폐율↔가로성능)을 손으로 체험. (결과 섹션)

## 6. 이미지 처리

- 렌더(매스/도시단면): 풀블리드, 상하 약한 그라데이션으로 텍스트 가독 확보
- 다이어그램(.ai→SVG): 배경 투명, 선 색을 `--ink` 단색 + 명도 단계(`--ink-70`/`--ink-40`)로 통일(원본 색 난립 방지)
- 21단지 사진: 동일 비율(4:3) 크롭, 약한 톤 통일(채도 -10%) → 카드 일관성
- AI 분석 이미지(세그멘테이션): 원본/마스크 나란히 비교(before/after 슬라이드)

## 8. 접근성 / 기본기

- 본문 대비 AA 이상(`--ink` on `--paper` = 충분)
- 무채색이라 본문/배경 대비는 자연히 충분(`--ink` on `--paper`)
- 포커스 링: `--ink` 2px outline + 1px offset
- 이미지 alt 필수, 다이어그램은 캡션으로 의미 보강
- 시맨틱 마크업(`<section> <figure> <table>`)

## 11. 디자인 무드 (Mood) — 키워드 번역

> 레퍼런스: 사용자 Pinterest 보드(`2026_졸업설계`) + 키워드 9개.
> ※ 보드는 로그인 필요로 직접 열람 못 함 → 핀을 캡처/설명해 주면 무드를 더 정밀하게 맞춤.

무드 한 줄: **"분석 노트처럼 정밀한 건축 도면" — Blueprint × Code Editor × AI Overlay.**
미니멀 도면체(§0)를 기반으로, 키워드의 *분석·코딩·AI* 결을 얹어 **"연구하는 화면"**으로 만든다.

### 키워드 → 화면 언어

| 키워드 | 화면에서의 번역 |
|---|---|
| **건축** | 도면 헤어라인, 단면/평면 선, 치수선(dimension line)·지시선, 모눈 |
| **깔끔한** | 넓은 여백, 무채색, 1px 선, 정렬된 그리드. 장식 0 |
| **분석적인** | 데이터 라벨, 좌표 눈금, 측정 캡션(`Fig.`/`H=`/`α=`), 작은 수치 주석 |
| **코딩** | mono 폰트 라벨, 키-값 표기(`BCR: 17.3%`), 코드블록형 캡션, 터미널풍 readout |
| **ai** | 세그멘테이션 마스크 오버레이(외곽선/도트), bounding box, 신뢰도(%) 태그 |
| **아파트** | 매스 footprint 패턴, 주동 반복 모듈, 단지 배치 다이어그램 |
| **보행자** | 눈높이(1.6m) 기준선, 시선 cone, sight-line, 휴먼 실루엣 스케일바 |
| **시각** | 앙각 호(arc), SVF 반구 다이어그램, 뷰코리더, 시야각 부채꼴 |
| **패턴** | 모눈·해칭(hatch)·도트 그리드, 매스 반복 타일링, 등고/리듬 반복 |

### 무드를 만드는 5가지 시그니처 디바이스
1. **치수선·지시선** — 이미지/다이어그램에 얇은 치수선과 지시선으로 수치를 달아 "분석 도면"처럼.
2. **모노스페이스 주석(readout)** — 화면 곳곳에 작은 mono 텍스트로 좌표·수치(`x:127 z:1.6m  α=51.8°`)를 흘려 코딩/분석 느낌.
3. **AI 세그멘테이션 오버레이** — 21단지 사진 위에 흰/검 외곽선 마스크 + 라벨(`sky 0.41` `building 0.39`)을 얹어 컴퓨터비전 톤.
4. **해칭·도트 패턴** — 면을 색으로 칠하지 않고 해칭선/도트로 채워 도면 질감(무채색 강조의 핵심 도구).
5. **시각 다이어그램** — 앙각 호·SVF 반구·시선 cone을 1px 선으로 그려 "보행자 시각" 메시지를 반복 모티프로.

### 톤의 균형 (과하지 않게)
- 베이스는 어디까지나 **깔끔한 미니멀 도면**. 위 디바이스는 **양념**으로, 섹션당 1~2개만.
- 분석/코딩 readout은 **진짜 데이터**(연구의 실제 수치)로 채운다 — 가짜 장식 텍스트 금지.
- AI 오버레이는 03b(사례분석) 섹션에 집중, 다른 섹션은 절제.

> 실제 코드: `site/index.html` · `site/css/style.css` · `site/js/main.js`

### 헤더 (고정 · 풀폭, 구분선 없음)

- 우: **언어 토글(한글 / ENG)** — 김다인 대신 상시 표시. 검은 테두리 세그먼트, CLOSE 위계, 활성 측 먹 채움

- **언어 토글(한글 / ENG)**: 우측에 상시 표시(김다인 대체). 검은 테두리 세그먼트, CLOSE와 동일 위계(Space Grotesk 500·대문자·자간 0.2em), 활성 측은 먹 채움
- 전환은 `data-ko`/`data-en` 텍스트 스왑 + 메뉴 재빌드(MENU의 kr/en) + localStorage(`sg-lang`). 기본 ko

### 메뉴 (좌측 드로어)
- 폭 `min(480px, 86vw)`, 좌측에서 슬라이드. **반투명**(paper 76% + blur 8px)으로 뒤 표지가 비침
- 나머지 화면은 그대로 유지. 바깥 클릭 / ESC 로 닫힘
- **GNB 위계**(동일 정렬선): 대제목 20px Medium 500 · ink / 소제목 16px Light 300 · ink-70(옅음)
- **소분류**: 대제목 hover 시 아래로 펼쳐짐(max-height). **대제목 기준 왼쪽 정렬**(들여쓰기 2.8rem). hover 시 **이동 없이** 굵기↑(300→400)·진한 색(ink). 텍스트만(불릿 없음). 터치 화면 상시 노출
- 항목/소분류 데이터(한/영)는 `main.js`의 `MENU` 배열에서 일괄 관리

### 표지 본문 — 키네틱 스카이라인 (skyline.js)
- 캔버스가 **화면 최상단(top:0, 헤더 뒤)부터** 푸터 위까지 차지 → 단어가 **헤더 위에서부터** 내려옴 (높이 = JS가 `cover − footer`)
- 컨셉(핀터레스트 "Berlin in Typography" 참고): **아파트 관련 '단어'가 천천히 떨어져 쌓이며 한국 아파트 단지 입면 완성**. 단어에 박스 없음(순수 글자). 풀: 아파트·주동·세대·발코니·용적률·건폐율·인동간격·판상형·탑상형 등
- 모델 = **건물 = (앞=짙고 낮음) + 왼쪽으로 오프셋된 (옅고 높음) 타워 적층**(`SITES[i].towers=[[x0,x1,h,c]…]`). 옅은 색이 **옆(왼쪽 띠) + 위(캡)** 로 솟음 (PDF의 오프셋 겹침)
- **옅은(뒤) 타워는 건물 너비만큼 온전한 직사각형으로 생성·낙하**. 앞(짙은) 타워에 가려지는 단어는 **draw 시 `coverY` 판정으로 안 그림**(evenodd 클립의 삼중겹침 비침 버그 제거). 타워는 자기 컬럼만 클립(좌우 직선)
- **3톤(PDF)**: 0 진함 `#4d4d4d` / 1 중간 `#9a9a9a` / 2 연함 `#cfcfcf`
- **건물 줄세우기 낙하**: 타워를 **색(진→연) → 같은 색 안 무작위 순서**로 줄세워 `towerStart = rank·(fallMs·0.22)`로 시차 → 앞 건물이 ~1/4 내려오면 다음 건물. 진한색 꼬리를 물고 다음 색이 이어짐(끝물→첫물)
- **건물 안 단어 펼침**: `delay = towerStart + rng·(fallMs·0.50)` → 한 건물 안 단어들이 넓게 퍼져 내려와 매스 모양이 예측되지 않음
- **소멸도 줄세우기(빌드 절반 길이)**: 연한색 먼저 → 건물별 순차 + 단어 펼침 (`outDelay = outRank·(TOWER_STEP·0.5) + rng·(WORD_SPREAD·0.5)`) → 내려올 때처럼 길게 늘어지며 사라짐
- **속도 변화**: 처음(진한색) 빌드 ×0.7, 마지막(진한색) 소멸 ×0.6
- **글자 크기 다양**: 타워별(baseFS×0.72~1.42) + 단어별(×0.86~1.16) 2중 변주
- 시작은 **화면 밖(위)**(spawnDist=H+70). SPEED 0.34, out ×2.4
- **중앙 정렬**: 각 단지의 전체 타워 바운딩박스(minX~maxX)를 계산해 `shift = 0.5 − (minX+maxX)/2`로 화면 중앙 이동(런타임, SITES 좌표는 유지)

### 푸터 정렬
- 텍스트 **세로 중앙 정렬**(`.cover__foot`·`.cover__col` align-items: center)
